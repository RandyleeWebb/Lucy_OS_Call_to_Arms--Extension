# NovaShell UI Specification  
_An advanced, touch‑first Windows‑style desktop environment_

---

## 1. Concept overview

**Goal:**  
Create a desktop UI that feels like Windows in power and familiarity, but behaves like a modern mobile OS: fluid animations, gesture‑driven navigation, and adaptive layouts for mouse, touch, pen, and gamepad.

**Core ideas:**

- **Desktop as a “space hub”** instead of a static wallpaper.
- **Windows as “cards”** that can dock, stack, or swipe like mobile screens.
- **Global gesture layer** for navigation, multitasking, and quick commands.
- **Unified control center** replacing scattered system trays and flyouts.

---

## 2. Design principles

- **Touch‑first, input‑agnostic:**  
  Every interaction must work with touch, mouse, keyboard, pen, and controller.

- **Spatial consistency:**  
  Same gestures always do the same thing (e.g., swipe from left = app switcher).

- **Progressive complexity:**  
  Simple for casual users, deep power features discoverable but not noisy.

- **High information density, low visual noise:**  
  Clear hierarchy, strong typography, subtle chrome.

- **Live, not static:**  
  Surfaces update in real time (widgets, live tiles/cards, notifications).

---

## 3. Global layout

### 3.1 Primary regions

- **Top system bar**
  - **Left:** App menu button, workspace switcher.
  - **Center:** Clock, quick status (network, battery, profile).
  - **Right:** Global search, notifications icon, control center icon.

- **Center workspace**
  - Multi‑desktop “spaces” (like virtual desktops) with:
    - Freeform windows.
    - Split views.
    - Fullscreen apps.
  - Background can host:
    - Widgets.
    - Pinned app cards.
    - Live system panels (e.g., performance, media).

- **Bottom dock / task rail**
  - Adaptive:
    - **Desktop mode:** Horizontal dock with icons + labels on hover.
    - **Tablet mode:** Larger icons, centered, with gesture handle.
  - Shows:
    - Pinned apps.
    - Running apps (with progress indicators).
    - System “shelf” (downloads, media, clipboard).

---

## 4. Navigation model

### 4.1 Core navigation concepts

- **Workspaces:**  
  Named sets of windows (e.g., “Work”, “Gaming”, “Focus”).

- **Stacks:**  
  Grouped windows of the same app or related apps (like tab groups, but OS‑level).

- **Cards:**  
  Visual representation of windows in overview, app switcher, and history.

### 4.2 Navigation entry points

- **Global search (⌘+Space / Win+Space / tap icon)**
  - Apps, files, settings, commands, web.
  - Inline actions (e.g., “toggle dark mode”, “connect Bluetooth”).

- **Workspace switcher**
  - Horizontal strip of workspaces at top.
  - Swipe left/right or scroll to change.
  - Long‑press/right‑click for workspace options.

- **App switcher**
  - Grid or carousel of app cards.
  - Filter by workspace, app type, or time used.

---

## 5. Gesture system

### 5.1 Edge gestures

- **Swipe from left edge**
  - Open app switcher (current workspace).
  - Continue dragging to “throw” app to another workspace.

- **Swipe from right edge**
  - Open control center (quick toggles, sliders, media, devices).

- **Swipe from bottom edge**
  - Reveal dock/task rail if hidden.
  - Long swipe: open global overview (all workspaces + apps).

- **Swipe from top edge**
  - App‑specific menu or command palette.
  - In fullscreen apps: show window controls and title.

### 5.2 In‑surface gestures

- **Three‑finger swipe left/right**
  - Switch workspace.

- **Three‑finger swipe up**
  - Open overview (all apps in current workspace).

- **Three‑finger swipe down**
  - Minimize all windows (show background widgets).

- **Pinch in**
  - Zoom out to workspace overview (all workspaces).

- **Pinch out**
  - Focus current app fullscreen.

---

## 6. Window and layout behavior

### 6.1 Window states

- **Floating:** Freeform, resizable, with subtle shadow and rounded corners.
- **Docked:** Snapped to edges or grid (e.g., 1/2, 1/3, 2/3 layouts).
- **Stacked:** Multiple windows layered with a tab‑like header strip.
- **Fullscreen:** Chrome‑less, gesture‑controlled system UI.
- **PIP (Picture‑in‑Picture):** Small always‑on‑top window for media or tools.

### 6.2 Layout templates

- **Quick layouts**
  - 50/50 split.
  - 33/33/33 triple.
  - 25/25/25/25 quad.
  - Golden‑ratio split (approx 38/62).

- **Layout picker**
  - Triggered by:
    - Dragging window to screen edge/corner.
    - Clicking “Layout” button on title bar.
  - Shows live preview of possible arrangements.

---

## 7. Visual language

### 7.1 Color and depth

- **Layered depth**
  - Background: soft blur, low contrast.
  - Windows: medium contrast, subtle elevation.
  - Overlays (menus, dialogs): higher elevation, stronger shadow.

- **Dynamic theming**
  - System accent color derived from wallpaper or user choice.
  - Light, dark, and high‑contrast modes.
  - Per‑workspace accent overrides (optional).

### 7.2 Typography and iconography

- **Typography**
  - Clear, geometric sans‑serif.
  - Large, readable titles; compact body text.
  - Consistent scale steps (e.g., 12, 14, 16, 20, 24, 32).

- **Icons**
  - Outline + filled variants.
  - Consistent stroke width.
  - Animated transitions for key actions (launch, close, pin).

---

## 8. Control center and system surfaces

### 8.1 Control center

- **Sections**
  - Connectivity: Wi‑Fi, Bluetooth, VPN, hotspot.
  - Device: Brightness, volume, output devices.
  - System: Focus mode, Do Not Disturb, performance profile.
  - Media: Current playback, output routing.
  - Quick actions: Screen record, cast, screenshot, night light.

- **Behavior**
  - Swipe from right edge or click icon.
  - Drag down to expand into full “system panel” with more details.

### 8.2 Notifications

- **Notification stream**
  - Right side panel, grouped by app and time.
  - Inline actions (reply, archive, mute app).

- **Focus modes**
  - Presets: Work, Personal, Gaming, Sleep.
  - Rules: time‑based, app‑based, device‑based.

---

## 9. Widgets and background

### 9.1 Widget system

- **Placement**
  - On desktop background.
  - In dedicated “Widget space” (swipe down with four fingers).

- **Types**
  - Info: weather, calendar, tasks, system stats.
  - Controls: smart home, media, quick scripts.
  - App‑linked: email, messaging, project dashboards.

### 9.2 Interaction

- **Tap:** Open associated app or expand widget.
- **Long‑press/right‑click:** Resize, configure, pin/unpin.
- **Drag:** Reorder, snap to grid.

---

## 10. Input modes

### 10.1 Mouse and keyboard

- **Hover states** for affordances (resize, drag, close).
- **Keyboard shortcuts**
  - Win+1..9: Launch/switch apps on dock.
  - Win+Tab: Overview.
  - Win+Ctrl+←/→: Switch workspace.
  - Win+Shift+S: Screenshot tools.
  - Ctrl+Space: Command palette.

### 10.2 Touch

- Larger hit targets, increased spacing in tablet mode.
- Gesture hints (subtle handles, edge glows).

### 10.3 Pen

- **Pen workspace**
  - Quick notes.
  - Screen sketch.
  - Whiteboard mode.

- **Pen shortcuts**
  - Double‑tap: Screenshot + annotate.
  - Button: Open notes or command palette.

---

## 11. Accessibility

- **High‑contrast themes** with strict color contrast.
- **Scalable UI density** (compact, comfortable, large).
- **Full keyboard navigation** with visible focus rings.
- **Screen reader‑friendly structure** for all system surfaces.
- **Reduced motion mode** to minimize animations and parallax.

---

## 12. Animation and motion

- **Principles**
  - Fast, responsive (150–250 ms for most transitions).
  - Purposeful: motion should explain state changes.
  - Interruptible: user input always wins.

- **Examples**
  - Window open/close: scale + fade with slight elevation change.
  - Workspace switch: parallax slide of background and windows.
  - Overview: windows shrink into cards with staggered timing.

---

## 13. Implementation notes (high‑level)

- **Rendering**
  - GPU‑accelerated compositor.
  - Support for blur, shadows, rounded corners, and live transforms.

- **Layout engine**
  - Constraint‑based or grid‑based layout for windows and widgets.
  - Saved layouts per workspace.

- **Gesture recognizer**
  - Unified gesture layer above windows.
  - Conflict resolution between app gestures and system gestures.

- **Extensibility**
  - Widget API.
  - App integration hooks (badges, actions, deep links).
  - Theme packs and icon packs.

---

## 14. Example user flows

### 14.1 Creating a focused workspace

1. **Open workspace switcher** from top bar.
2. **Create new workspace** named “Writing”.
3. **Open writing app** and browser.
4. **Use layout picker** to set 70/30 split.
5. **Enable Focus mode** from control center.
6. **Pin writing stats widget** to background.

### 14.2 Multitasking with gestures

1. In fullscreen app, **swipe from left edge** to open app switcher.
2. Drag another app card to right side to create split view.
3. **Three‑finger swipe up** to overview, drag a third app into stack.
4. **Pinch out** to focus on the main app again.

---

## 15. Summary

NovaShell is a Windows‑class desktop with mobile‑grade fluidity:  
- Gesture‑driven navigation.  
- Workspace‑centric organization.  
- Deep but discoverable power features.  
- A visual language tuned for both productivity and touch.

This spec is meant as a foundation you can extend into actual wireframes, prototypes, and implementation details.
