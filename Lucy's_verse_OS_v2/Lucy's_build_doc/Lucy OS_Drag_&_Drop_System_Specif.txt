Lucy OS — Drag & Drop System Specification
Advanced ingestion pipeline for files, media, text, and objects inside a virtual desktop

1. Core Concept
Lucy does not run on the real Windows desktop.
She runs inside her own virtual desktop sandbox (Electron + Chromium surface).

Therefore:

Windows cannot drop files directly into Lucy.
Electron must intercept the drop event, and Lucy must decide what to do with the file.

This is the foundation of the entire system.

2. High-Level Flow
Code
User drops file → Electron captures → IPC → Lucy Frontend → Lucy OS Ingestion Engine → Library / Tools / Agents
Broken down:

User drags file over Lucy’s window

Electron receives OS-level drag event

Electron sends file metadata → Renderer via IPC

Lucy UI shows drop targets / animations

User releases file

Lucy OS Ingestion Engine classifies file

Lucy decides action (auto or Ask Lucy modal)

Lucy imports file into internal library

Lucy triggers agents (Audio, Vision, Document, etc.)

7. Electron → Lucy IPC Pipeline
Electron Main
Captures OS drop event:

js
mainWindow.webContents.send("file-dropped", filePaths)
Renderer
Receives event:

js
ipcRenderer.on("file-dropped", (event, files) => {
    lucy.handleDrop(files)
})
Lucy OS
Runs ingestion engine:

ts
handleDrop(files) {
    const classified = classify(files)
    const actions = suggestActions(classified)
    showAskLucyModal(classified, actions)
}

10. Summary (One Sentence)
Lucy’s drag‑and‑drop system is a virtual-desktop ingestion pipeline where Electron captures the drop, Lucy classifies the file, chooses an action, imports it into her internal library, and triggers the correct agents to process it.
