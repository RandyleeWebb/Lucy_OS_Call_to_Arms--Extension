Lucy OS Swarm Dashboard — Full UI Layout (Markdown Spec)
1. Top-level screen structure
Screen name: Swarm Operations Dashboard  
Primary goal: Real‑time command center for Lucy’s cognitive loop, events, agents, and executions.

Layout grid (desktop, 16:9, touch‑friendly):

Top bar (64px–80px height)

Main body split:

Left column (25–30% width) — Awareness & Events

Center column (40–50% width) — Tasks, Plans, Decomposition

Right column (25–30% width) — Agents, Execution, Memory

Bottom strip (48–64px height) — Status rail & quick actions

2. Top bar — Global control & context
Region: Full width, fixed at top.

Contents:

Left section

Logo / Title: Lucy OS · Swarm Dashboard

Mode indicator: LIVE / PAUSED (toggle)

Workspace selector: Global · Project · System · User

Center section

Cognitive loop status pill:

Text: Loop: Running (every 5s)

Subtext: Last cycle: 1.2s · Next in: 3.8s

Loop phase indicator (stepper):  
Events → Intake → Planner → Decomposer → Swarm → Execution → Feedback → Memory → Replan

Current phase highlighted, others dimmed.

Right section

Global search: Search tasks, events, agents…

Profile / persona: Emma (Sovereign Coordinator)

System icons: notifications, settings, theme toggle.

3. Left column — Awareness & events (W.A.T.C.H)
3.1 Panel: Real-time awareness overview
Title: W.A.T.C.H · Awareness  
Position: Top of left column.

Contents:

Mini tiles (2×3 grid):

OS: CPU, memory, disk (small sparkline)

Apps: active windows count, foreground app

Network: up/down, latency

Browser: active tabs, tracked domains

Agents: active vs idle

Alerts: count by severity

Each tile is tap/clickable → opens detailed view.

3.2 Panel: Event stream
Title: Event Bus · Live Stream  
Position: Below awareness panel, scrollable.

Layout:

Filter bar (top):

Source chips: OS, Apps, Network, Docker, Git, Email, Discord, Calendar, Lucy

Severity filter: All · Info · Warn · Critical

Toggle: Only actionable events

Event list (timeline):

Each row:

Timestamp

Icon + source (e.g., Docker, Filesystem)

Event name (e.g., docker.container.started)

Priority badge (1–5)

Suggested action (small pill, e.g., index_document)

Status: queued → converted_to_task → ignored

Tap/click event → side drawer with full JSON payload and history.

7. Mobile / touch behavior & gestures
Even though this is a “desktop” dashboard, it should behave like a mobile‑grade UI.

7.1 Global gestures
Swipe from left edge: Open navigation drawer (switch dashboards: Swarm, DevOps, Logs, Settings).

Swipe from right edge: Open notifications & alerts panel.

Pinch out on center column: Expand current task to full‑screen “focus mode”.

Three‑finger swipe left/right: Switch between:

Overview · Tasks · Agents · Execution

7.2 Panel interactions
Panels are dockable cards:

Long‑press title bar → drag to rearrange columns.

Double‑tap title → maximize panel in place.

Swipe down on panel → minimize to chip in bottom strip.
